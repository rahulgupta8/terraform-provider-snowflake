# snowflake_contact (Resource)

Resource used to manage contact objects. For more information, check [contact documentation](https://docs.snowflake.com/en/user-guide/contacts-using).

## Example Usage

```terraform
resource "snowflake_contact" "example" {
  name    = "example_contact"
  email   = "contact@example.com"
  comment = "Example contact for notifications"
}
```

-> **Note** If a field has a default value, it is shown next to the type in the schema.

## Schema

### Required

- `name` (String) Specifies the identifier for the contact; must be unique for the account. Due to technical limitations (read more [here](../guides/identifiers_rework_design_decisions#known-limitations-and-identifier-recommendations)), avoid using the following characters: `|`, `.`, `"`.

### Optional

- `comment` (String) Specifies a comment for the contact.
- `email` (String) Email address for the contact.

### Read-Only

- `fully_qualified_name` (String) Fully qualified name of the resource. For more information, see [object name resolution](https://docs.snowflake.com/en/sql-reference/name-resolution).
- `show_output` (List of Object) Outputs the result of `SHOW CONTACTS` for the given contact. (see [below for nested schema](#nestedobjatt--show_output))
